## Bugs Fixed During Day 14

### Bug 1: Non-JSON server stdout lines corrupted client stream

**Symptom:** Node.js MCP servers (filesystem, GitHub) occasionally write startup log lines like `Server running on stdio` to stdout before sending their first JSON message. The proxy was forwarding these raw text lines to the client, corrupting its JSON-RPC parser.

**Fix:** `src/mcp_debugger/proxy/stdio_proxy.py` — `_server_to_client_loop()` now checks `json.loads(line)` before forwarding. Non-JSON lines are:
- Printed to stderr as `[mcp-debugger] server log: ...`
- Stored in the new `server_logs` table in the database
- **Never forwarded to the client**

### Bug 2: Windows short path rejection by filesystem server

**Symptom:** `tempfile.TemporaryDirectory()` on Windows may return a path with 8.3 short names (e.g. `C:\Users\SUSHAN~1\...`). The filesystem server performs a strict prefix check against the allowed directories and rejects short paths.

**Fix:** `scripts/test_real_servers.py` — resolve tmpdir with `Path.resolve()` before using it in commands and tool arguments.

## CI Strategy

For automated CI (GitHub Actions), only the **filesystem** server is run on every push because:
- It has no external dependencies (no API keys)
- It is fast (~20s total including npm install)
- It is the most commonly used MCP server

Other servers (GitHub, Fetch, Slack, PostgreSQL) should be run manually or in a nightly job with secrets configured.

See `.github/workflows/ci.yml` for the CI configuration.

## Running Compatibility Tests Manually

```bash
# Filesystem (no deps)
python scripts/test_real_servers.py --server-name filesystem

# GitHub (requires token)
GITHUB_PERSONAL_ACCESS_TOKEN=<your-token> \
  python scripts/test_real_servers.py --server-name github

# All servers (will skip those missing env vars)
python scripts/test_real_servers.py --all --output docs/compat_report.md

# Custom server
python scripts/test_real_servers.py \
  --server "npx -y @modelcontextprotocol/server-filesystem /tmp" \
  --startup-wait 15 --step-timeout 20
```
